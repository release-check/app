# RFC 0001: MVP

## Goal

Build the smallest ReleaseCheck that proves the core idea:

1. Search a track or release.
2. Find candidates from multiple sources.
3. Decide whether candidates refer to the same music.
4. Return platform availability with confidence and evidence.

## MVP Endpoints

- `GET /search?q=`
- `GET /availability?artist=&track=`
- `GET /resolve?url=`
- `POST /batch`

## MVP Demo Dataset

Start with a curated seed dataset of independent and underground releases, then add platform adapters one by one.

The demo should prioritize visible matching evidence over broad but shallow coverage.
